# 🐍 Python 실전 코딩 핵심 노트

파이썬을 공부하고 코딩을 진행하며 새롭게 알게 된 핵심 개념과 주의할 점들을 정리한 문서입니다.

---

## 1. 객체 지향과 패키지 관리 (OOP & Modules)

### 1.1 클래스 상속과 객체 초기화
* 클래스의 상속이 발생할 때, 상속받은 자식 클래스는 반드시 인스턴스(객체)의 초기화(`__init__`)가 필요합니다.
* 초기화를 하지 않으면 부모 클래스의 `__new__`(중복 방지) 메서드 로직 때문에 이미 인스턴스가 존재하는 것으로 간주되어 작업이 종료될 수 있습니다.

### 1.2 패키지 인식자 `__init__.py`
* 특정 폴더에 `__init__.py`가 존재하면 파이썬은 해당 폴더를 일반 폴더가 아닌 **패키지**로 인식합니다.
* **접근 제어 및 Re-export:** 사용자에게 어떤 함수나 클래스를 보여줄지 정리할 수 있습니다. 예를 들어, `schemas` 폴더 내에 `bank_inquiry_schema.py`가 있을 때, 그 안의 세부 코드를 숨기고 `__init__.py`에 `add` 함수만 import 해두면 `from schemas import add`로 간단하게 다시 내보내기(Re-export)가 가능합니다.

### 1.3 타입 검사
* `isinstance(객체, 클래스 타입)`를 사용하면 해당 객체가 특정 클래스 타입인지 판별하여 `True` 또는 `False`를 반환합니다.

## 2. 자료구조 다루기 (List & Set)

### 2.1 리스트(List) 요소 삭제 시 주의점
* 리스트를 순회하며 `pop`이나 `remove`를 사용할 때 절대 주의해야 합니다. 해당 요소가 삭제되면서 **뒤의 요소들이 한 칸씩 앞으로 당겨져 인덱스(index)가 줄어들기 때문**입니다.

### 2.2 집합(Set) 요소 삭제
* `remove()`: 지우려는 요소가 집합에 없으면 **에러가 발생**합니다.
* `discard()`: 지우려는 요소가 없어도 **에러가 발생하지 않습니다.**

### 2.3 리스트 중복 제거 및 집합 연산
* 중복을 제거하고 싶을 때는 `set`으로 묶었다가 다시 `list`로 변환하는 방식을 활용합니다. (예: `list(set(a))`)
* `set` 상태(`{}` 형태)에서는 `|`, `&`, `-` 연산자를 활용해 각각 합집합, 교집합, 차집합을 쉽게 계산할 수 있습니다.

## 3. 정렬과 리스트 컴프리헨션 (Sorting & Comprehension)

### 3.1 `sort()`와 `sorted()`의 리턴값 차이
* **`a.sort()`**: 원본 리스트 `a`를 직접 변환합니다. 원본을 변환시키는 메서드이므로 **리턴값이 없습니다** (`None`).
* **`sorted(a)`**: 원본은 그대로 두고 정렬된 새로운 복사본을 생성해서 리턴합니다. 따라서 `a = sorted(a)` 형태로 받아주어야 원본 변수에 정렬된 결과가 덮어씌워집니다.

### 3.2 다중 조건 정렬
* 리스트 안에 리스트나 튜플이 존재할 때 `lambda`를 활용하여 정렬 조건을 지정할 수 있습니다.
* **1개의 조건:** `a.sort(key=lambda x: x[1])` 또는 `a.sort(key=len)`
* **2개 이상의 다중 조건:** `a.sort(key=lambda x: (x[1], x[0]))`
    * 1순위 조건(`x[1]`), 2순위 조건(`x[0]`) 순으로 적용됩니다.
    * **내림차순**이 필요한 조건 앞에는 `-`를 붙입니다. (예: `-x[1]`)
    * **주의:** 2가지 이상의 조건을 사용할 때는 반드시 괄호로 묶어 **튜플 `()` 형태**로 작성해야 합니다. 그렇지 않으면 두 번째 인자가 `key`에 제대로 입력되지 않아 `SyntaxError`가 발생합니다.

### 3.3 리스트 컴프리헨션(Comprehension)의 구조
* **if문 필터링:** `[결과값 for문 if문]` ➝ `if`문이 조건 필터 역할을 합니다.
* **if-else문 사용:** `[결과값 if문 else문 for문]` ➝ 결과값부터 `else`문까지를 하나의 결과 덩어리로 인식하기 때문에, `for`문 앞으로 통째로 위치해야 합니다.

## 4. 문자열 제어 (String Methods)

### 4.1 `strip()`의 정확한 동작
* `strip("abc")`는 "abc라는 문자열을 통째로 제거"하는 것이 아닙니다. 문자열의 **양 끝(접두/접미)에 a, b, c 중 하나라도 있으면 모두 제거**한다는 의미입니다.

### 4.2 Iterable을 문자열로 합치기
* 리스트 같은 Iterable 객체 안의 문자열들을 하나의 문자열로 결합하고 싶을 때는 `join()` 함수를 사용하는 것을 잊지 마세요. (예: `"".join(my_list)`)

### 4.3 문자열 확인 메서드
* `isdigit()`은 문자열에서만 사용 가능한 메서드입니다. 이 외에도 `isalpha`, `isalnum`, `islower`, `isupper` 등이 있습니다.
* **숫자 판별의 차이:**
    * `isdecimal()`: `0-9`와 같은 일반적인 아라비아 숫자 형태만 인정합니다.
    * `isdigit()`: 거듭제곱(²) 등 '숫자처럼 생긴 글자'도 인정합니다.
    * `isnumeric()`: 분수(½), 한자 숫자 등 숫자를 의미하는 모든 문자를 포괄적으로 인식합니다.

## 5. 제어문과 기타 개념

### 5.1 `for-else` 문법
* `for`문을 도는 동안 `break`를 만나지 않고 루프가 끝까지 정상 종료되면, 이어서 `else` 블록의 코드를 수행합니다.

### 5.2 메서드(Method)와 함수(Function)의 차이
* **메서드:** `instance.upper()`와 같이 `객체.기능()` 형태로 사용하며, 특정 자료형(예: 문자열)에 종속된 전용 기능입니다.
* **함수:** `len(my_string)`과 같이 `기능(객체)` 형태로 사용하며, 특정 객체에 얽매이지 않고 여러 자료형에 두루 쓰일 수 있는 범용적인 기능입니다.
